# 手动启动前端 - 详细步骤

## 🔍 问题说明

如果批处理脚本闪退，可能是：
1. 命令执行失败但没有显示错误
2. 路径问题
3. Node.js环境问题

## ✅ 手动启动步骤（推荐）

### 步骤1: 打开命令提示符

**方法1**: 
- 按 `Win + R`
- 输入 `cmd`
- 按回车

**方法2**:
- 按 `Win + X`
- 选择"Windows PowerShell"或"命令提示符"

### 步骤2: 进入前端目录

在命令提示符中输入：

```bash
cd F:\VabHub项目\VabHub\frontend
```

按回车，应该看到提示符变成：
```
F:\VabHub项目\VabHub\frontend>
```

### 步骤3: 检查Node.js

输入：
```bash
node --version
```

应该显示版本号，比如：`v25.1.0`

如果没有显示，说明Node.js未安装或未添加到PATH。

### 步骤4: 启动前端服务

输入：
```bash
npx vite --host 0.0.0.0 --port 5173
```

按回车，应该看到类似输出：
```
VITE v5.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/
➜  Network: http://192.168.x.x:5173/
```

### 步骤5: 访问前端

在浏览器中打开：http://localhost:5173

**重要**: 保持命令提示符窗口打开，不要关闭！

---

## 🐛 如果步骤3失败（Node.js未找到）

### 安装Node.js

1. 访问：https://nodejs.org/
2. 下载LTS版本（推荐）
3. 安装时选择"Add to PATH"
4. 重启命令提示符
5. 再次运行 `node --version` 验证

---

## 🐛 如果步骤4失败（启动失败）

### 错误1: "npx不是内部或外部命令"

**解决**:
```bash
npm install -g npm@latest
```

### 错误2: "端口5173被占用"

**解决1**: 使用其他端口
```bash
npx vite --host 0.0.0.0 --port 5174
```

**解决2**: 结束占用进程
```bash
# 查找占用进程
netstat -ano | findstr ":5173"

# 结束进程（替换PID为实际进程ID）
taskkill /PID <进程ID> /F
```

### 错误3: 网络错误

**解决**:
1. 检查网络连接
2. 尝试清除npm缓存：
   ```bash
   npm cache clean --force
   ```

2. 访问 http://localhost:5173

## 🆘 如果仍然无法启动

### 收集信息

请提供以下信息：

1. **Node.js版本**: 
   ```bash
   node --version
   ```

2. **npm版本**: 
   ```bash
   npm --version
   ```

3. **完整错误信息**: 
   - 复制命令提示符中的完整输出
   - 包括所有错误信息

4. **当前目录**: 
   ```bash
   cd
   ```

5. **文件是否存在**: 
   ```bash
   dir package.json
   ```
