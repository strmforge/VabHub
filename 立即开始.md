# 🚀 立即开始：第一步行动

## 建议：从认证系统开始

### 为什么？
1. ✅ **代码框架已就绪** - User模型、security.py、auth.py都已创建
2. ✅ **依赖已配置** - requirements.txt包含所有必要依赖
3. ✅ **这是基础** - 没有认证，其他功能无法使用
4. ✅ **快速验证** - 可以快速验证系统是否能正常运行

---

## 📋 第一步：测试认证系统（30分钟）

### 1. 安装依赖（5分钟）

```bash
cd VabHub
pip install -r requirements.txt
```

### 2. 初始化数据库（5分钟）

```bash
cd backend
python -c "from app.core.database import init_db; import asyncio; asyncio.run(init_db())"
```

应该会创建 `vabhub.db` 文件。

### 3. 启动后端（5分钟）

```bash
python main.py
```

或者：

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

访问 http://localhost:8000/docs 查看API文档。

### 4. 测试API（10分钟）

#### 4.1 注册用户
在API文档中测试 `/api/v1/auth/register`：
```json
{
  "username": "testuser",
  "email": "test@example.com",
  "password": "test123456",
  "full_name": "Test User"
}
```

#### 4.2 登录
测试 `/api/v1/auth/login`：
- username: `testuser`
- password: `test123456`

应该返回token。

#### 4.3 获取用户信息
测试 `/api/v1/auth/me`，使用返回的token。

### 5. 前端联调（5分钟）

```bash
cd frontend
npm install
npm run dev
```

访问 http://localhost:5173/login，尝试登录。

## 🎯 如果遇到问题

### 问题1：数据库初始化失败
```bash
pip install aiosqlite
```

### 问题2：JWT错误
检查 `backend/app/core/config.py` 中的 `JWT_SECRET_KEY`

### 问题3：CORS错误
检查 `backend/app/core/config.py` 中的 `CORS_ORIGINS` 是否包含 `http://localhost:5173`
