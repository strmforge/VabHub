## 2. UI 设计原则

### 2.1 整体布局原则
- **标题区统一**：使用 `v-card-title` + `subtitle`，卡片主操作放在右上角
- **列表页结构**：顶部保留「统计摘要条 + 关键过滤器」，下方再是表格/卡片
- **一致的间距**：统一使用 `pa-4` 作为卡片内边距，`mb-4` 作为卡片间距
- **卡片样式**：统一使用 `rounded="xl"`、`elevation="1"` 等样式
- **按钮样式**：主操作按钮使用 `color="primary"`，次要按钮使用 `variant="text"`

### 2.2 状态标识原则
- **进行中**：使用 `color="primary"` + `v-icon="mdi-play-circle"`
- **已完成**：使用 `color="success"` + `v-icon="mdi-check-circle"`
- **失败**：使用 `color="error"` + `v-icon="mdi-alert-circle"`
- **风险**：使用 `color="warning"` + `v-icon="mdi-alert"`
- **暂停**：使用 `color="grey"` + `v-icon="mdi-pause-circle"`

### 2.3 空状态原则
- 统一使用 `v-alert` 组件展示空状态
- 提供明确的提示文本，说明当前状态
- 如有配置入口，提供直接跳转链接
- 示例：`暂无数据，请先配置下载器` + 「去配置」按钮

### 2.4 加载状态原则
- 数据加载时显示 `v-progress-circular` 或 `v-progress-linear`
- 表格/列表加载时使用 `v-skeleton-loader`
- 加载状态位置固定在内容区域，不影响页面布局

### 2.5 表单设计原则
- 统一使用 `variant="outlined"` 和 `density="compact"`
- 必填项使用红色星号标记
- 提供清晰的 `hint` 文本说明
- 表单分组使用 `v-divider` 或 `v-subheader` 分隔

### 2.6 数据可视化原则
- 进度条使用统一的颜色和高度
- 图表使用一致的配色方案
- 数值展示使用 `text-h6` 突出显示
- 统计卡片使用 `v-stat` 组件（如适用）

## 4. 代码规范

### 4.1 组件命名
- 使用 PascalCase 命名组件
- 组件名与文件名保持一致

### 4.2 样式管理
- 优先使用 Vuetify 内置样式类
- 自定义样式使用 scoped 属性
- 避免使用 !important 覆盖样式

### 4.3 响应式设计
- 使用 Vuetify 的网格系统实现响应式布局
- 针对不同屏幕尺寸优化组件显示
- 移动设备优先的设计思路

## 8. UI-FEEDBACK-FIX-1 修复记录

### 8.1 修复的问题

1. **HomeDashboard 最近活动时间线**：优化时间线布局，添加文字换行和最小宽度样式，确保文字完整显示
2. **PlayerWall 海报卡片**：移除 dense 属性，添加 g-4 间距，调整响应式断点，确保小屏幕上有适当间距
3. **MediaDetail 页面按钮区域**：添加 wrap="wrap" 和 gap="2" 属性，优化按钮间距，确保在小屏幕上合理换行
4. **系统健康卡片样式**：修改 SystemHealthCard 组件，添加 rounded="xl"、elevation="1" 和 pa-4 属性，统一卡片样式
5. **任务汇总卡片**：将任务汇总卡片中的数字改为 text-h4 字体大小和 font-weight-bold 样式，增强视觉突出度
6. **后台服务状态表格**：为表格行添加固定高度 40px，减小单元格内边距，增加信息密度

### 8.2 设计原则补充

- **响应式布局优先级**：确保在小屏幕设备上的良好显示效果
- **信息层级清晰化**：关键数字和状态信息应突出显示
- **间距一致性**：统一卡片和组件之间的间距，避免拥挤或松散
- **文字显示完整性**：确保所有文字信息在各种屏幕尺寸下都能完整显示或合理截断
