# KNOWN LIMITATIONS（已知限制 & 暂不处理事项）

## 1. 功能层面限制
- [ ] 非媒体资源只在下载管理中可见，缺乏专门的"其他资源"视图
- [ ] 某些高级功能（如 HR 风险检测）需要额外配置才能使用
- [ ] 部分 UI 组件可能存在样式不一致问题
- [ ] 部分功能仍在开发中，可能存在不完整的情况
- [ ] 部分 API 可能存在性能问题，需要进一步优化

## 2. 性能与规模
- [ ] 本版本未针对 10w+ 种子/媒体库进行压测，可能在极大规模下性能下降
- [ ] 大规模媒体库下，某些操作（如全站扫描）可能耗时较长
- [ ] 部分页面在数据量大时可能存在渲染延迟
- [ ] 某些后台任务（如 TTS 转换）可能占用较多资源

## 3. 部署与环境
- [x] 当前已完成 Docker/docker-compose 部署文档，这是官方推荐且唯一维护的部署方式
- [ ] 仅提供单机 docker-compose 模式，未提供 k8s Helm Chart
- [ ] 暂未提供多实例/多节点部署指导
- [ ] 日志/监控/备份工具集成需要用户自行安排
- [ ] 部分功能依赖外部服务（如 TMDB API、TTS 服务等）
- [ ] 某些功能可能需要特定的操作系统或环境配置
- [ ] 部分依赖可能存在版本兼容性问题

## 4. Docker 首跑场景下已知限制

### 首次启动限制
- [ ] 首次启动时，AI Orchestrator 默认关闭，需手动开启并配置外部 LLM
- [ ] 首次启动时，Telegram Bot 默认关闭，需手动配置 Token 并开启
- [ ] 首次启动时，部分高级功能（如 HR 风险检测）默认关闭，需手动开启
- [ ] 首次启动时，无预设用户，需通过 API 或命令行创建初始用户

### 配置建议
- 首次启动建议先只配置数据库和基础路径，PT 站点 / Bot / AI 可后补
- 配置 TMDB_API_KEY 以获取完整的影视元数据
- 如遇启动问题，建议先查看容器日志：`docker compose logs <service>`

## 5. AI & 外部服务
- [ ] AI Orchestrator 默认为 Dummy LLM，未配置真实 LLM 时所有 AI 页面仅作演示
- [ ] 站点 AI 适配通道严格限于站点结构解析，不提供通用对话功能
- [ ] 部分 AI 功能需要外部 LLM 支持才能正常工作
- [ ] AI 功能目前仅提供只读建议，不支持自动执行操作
