# 分步执行命令指南

## ⚠️ 重要提示

**不要把所有命令放在一行！** 每个命令需要单独执行。

---

## ✅ 正确的执行方式

### 方法1: 使用批处理脚本（推荐）

**双击运行**: `完整修复步骤.bat`

这个脚本会自动执行所有步骤。

### 方法2: 手动分步执行

#### 步骤1: 打开命令提示符

- 按 `Win + R`
- 输入 `cmd`
- 按回车

#### 步骤2: 进入前端目录

**只执行这一条命令**：

```bash
cd F:\VabHub项目\VabHub\frontend
```

按回车，等待执行完成。

#### 步骤3: 删除node_modules

**只执行这一条命令**：

```bash
rmdir /s /q node_modules
```

按回车，等待执行完成（可能需要几秒钟）。

#### 步骤4: 删除package-lock.json

**只执行这一条命令**：

```bash
del package-lock.json
```

按回车。

#### 步骤5: 安装依赖

**只执行这一条命令**：

```bash
npm install --legacy-peer-deps
```

按回车，**等待安装完成**（可能需要几分钟）。

#### 步骤6: 启动服务

**只执行这一条命令**：

```bash
npm run dev
```

按回车，等待启动完成。

## 📋 完整命令列表（逐条执行）

**重要**: 每次只复制一条命令，执行完成后再执行下一条！

```bash
# 第1条：进入目录
cd F:\VabHub项目\VabHub\frontend

# 第2条：删除node_modules（等待完成）
rmdir /s /q node_modules

# 第3条：删除package-lock.json
del package-lock.json

# 第4条：安装依赖（等待完成，可能需要几分钟）
npm install --legacy-peer-deps

# 第5条：启动服务
npm run dev
```

## 🐛 常见错误

### 错误1: "系统找不到指定的路径"

**原因**: 命令被合并在一行了

**解决**: 每次只执行一条命令

### 错误2: "无法删除node_modules"

**原因**: 文件被占用

**解决**: 
1. 关闭所有可能使用这些文件的程序
2. 或者重启电脑后再试

### 错误3: 安装依赖失败

**解决**: 使用国内镜像

```bash
npm config set registry https://registry.npmmirror.com
npm install --legacy-peer-deps
```
