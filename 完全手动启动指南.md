# 完全手动启动指南

## 🔍 如果所有批处理脚本都闪退

这说明可能是：
1. Node.js未安装
2. 环境变量未配置
3. 批处理脚本执行权限问题

## ✅ 解决方案：完全手动启动

### 方法1: 使用PowerShell（推荐）

#### 步骤1: 打开PowerShell

**方法1**:
- 按 `Win + X`
- 选择"Windows PowerShell"或"终端"

**方法2**:
- 按 `Win + R`
- 输入 `powershell`
- 按回车

#### 步骤2: 进入前端目录

在PowerShell中输入：

```powershell
cd F:\VabHub项目\VabHub\frontend
```

按回车，应该看到提示符变成：
```
PS F:\VabHub项目\VabHub\frontend>
```

#### 步骤3: 检查环境

```powershell
node --version
npm --version
```

如果显示版本号，说明环境正常。

如果显示"找不到命令"，说明Node.js未安装，请先安装Node.js。

#### 步骤4: 启动服务

```powershell
npx vite --host 0.0.0.0 --port 5173
```

按回车，等待启动完成。

#### 步骤5: 访问前端

在浏览器中打开：http://localhost:5173

## 🐛 如果Node.js未安装

### 安装Node.js

1. **访问官网**: https://nodejs.org/
2. **下载LTS版本**（推荐，稳定版）
3. **运行安装程序**
4. **重要**: 安装时勾选"Add to PATH"
5. **完成安装**
6. **重启命令提示符或PowerShell**
7. **验证安装**:
   ```bash
   node --version
   npm --version
   ```

## 🆘 如果仍然无法启动

### 检查1: Node.js是否真的安装了

```bash
where node
```

如果显示路径，说明已安装。
如果显示"找不到"，说明未安装或未添加到PATH。

### 检查2: 手动添加到PATH

如果Node.js已安装但找不到：

1. 找到Node.js安装目录（通常在 `C:\Program Files\nodejs\`）
2. 添加到系统PATH：
   - 右键"此电脑" -> 属性
   - 高级系统设置
   - 环境变量
   - 编辑Path
   - 添加Node.js安装目录
3. 重启命令提示符

### 检查3: 使用完整路径

如果PATH有问题，可以使用完整路径：

```bash
"C:\Program Files\nodejs\npx.cmd" vite --host 0.0.0.0 --port 5173
```
